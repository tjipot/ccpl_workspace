##CH14: 重载操作符与转换 OVERLOAD CONVERSION
CPP是一个系统, 这个系统已经定义了它的内置类型的操作符和自动转换, 这些设施使得程序员能编写出丰富的<u><strong>混合类型表达式</strong></u>.

CPP允许重定义操作符用于类类型对象间的操作, 使得类类型可以像内置类型那样地转换, 以及将一个类型的对象<u><strong>隐式地</strong></u>转换到另一个类型. (阿里巴巴与四十大盗的宝藏: "↑↓←→↖↗↘↙↕")

####14.1  重载操作符定义
重载操作符(函数)是有特殊名称的函数: 函数名为保留字"operator"+"操作符符号"; 重载操作符也具有返回类型和形参表; 编译器通过发现表达式中的操作符重载, 来调用相关的重载操作符(函数).

1.不可重载的操作符: "::", ".*", ":?", ".", "sizeof";

2.重载操作符必须具有一个类类型的操作数;

3.优先级和结合性是固定的;

4.不再具备短路求值特征;

5.可定义成类成员或非类成员(函数);

6.操作符重载和友元关系;

7.直接调用重载操作符.

######重载操作符的设计
1.不重载具有内置含义的操作符;

2.很多操作符对类对象没有意义;

3.复合赋值操作符: 举例, +=应该和+之后接=类似;

4.相等和关系操作符;

5.选择成员或非成员实现.

####14.10 小结
<u>重载操作符</u>和<u>类类型</u>转换都有助于更容易, 更自然地使用类型:

######重载操作符 OPERATOR OVERLOAD:
通过定义内置操作符的重载版本(←↑→↓↖↙↗↘↕触发机制), 可以为自定义类型(类类型或枚举类型)的对象定义和内置类型一样丰富的表达式集合;

重载操作符的操作数必须至少有一个是类类型或枚举类型, 且应与内置类型的操作符有同样书目的操作数, 结合性和优先级;

大多数重载操作符可以定义为类成员或非成员(普通)函数, 除了: 赋值操作符, 下标操作符, 调用操作符和箭头操作符, 必须为类成员;

"函数对象", 是重载了"operator()"调用操作符的类的对象, 通常用于定义谓词函数.


######类型转换 CONVERSION:
类也可以定义转换: 当一个类型的对象用在需要另一个不同类型对象的地方时, 转换会自动进行;

其它类型->类类型的转换: 接受单个形参, 未指定explicit的构造函数;

类类型->其它类型的转换: 重载操作符转换函数, 如: "operator int() const { return val; };", 必须为类成员, 无形参, 不定义返回值, 目标类型为int.
